# CourtIQ — Project Standards & Best Practices

## 1. Architecture Overview

```
pickleball-league/
├── index.html          # Single page app shell — all tabs
├── app.v3.js           # All JS logic (Firebase, auth, league, scoring)
├── style.css           # All styles (CourtIQ theme, components)
├── config.js           # Firebase API key (NOT committed — generated by Vercel)

├── .gitignore          # Excludes config.js
└── STANDARDS.md        # This file
```

**Stack:** Vanilla HTML/CSS/JS · Firebase (Firestore + Auth + Realtime DB) · Vercel

## 2. Security Standards

### API Keys
- ✅ Firebase API key stored in Vercel Environment Variables only
- ✅ `config.js` is in `.gitignore` — never committed
- ✅ Vercel build command generates `config.js` at deploy time
- ✅ Fallback key in `app.v3.js` is acceptable since Firestore rules protect data
- ❌ NEVER commit API keys, passwords, or secrets to git

### Firebase Security Rules
```
/leagues   — anyone can read, only auth users can write
/history   — anyone can read, only auth users can write
/groups    — only auth users can read/write their own document
/users     — auth users can read all, only write own document
/ratings   — auth users can read/write
```

### Authentication
- Google Sign-In via Firebase Auth
- `signInWithPopup` for desktop, `signInWithRedirect` for mobile
- Admin check: `currentUser?.email === ADMIN_EMAIL`
- Per-user data isolated by `currentUser.uid`

### Input Validation
- Score validation: min 11 pts, 2pt lead, no ties
- Email validation before saving groups
- Player count: even numbers only, min 4, max 20

## 3. Code Standards

### JavaScript Rules
```javascript
// ✅ DO — Use DOM methods for dynamic content with user data
const chip = document.createElement('div');
chip.textContent = userInput;  // Safe — no XSS
chip.addEventListener('click', () => handler(data));

// ❌ DON'T — Inline onclick with dynamic string values
element.innerHTML = '<div onclick="fn(\'' + variable + '\')">'; // BREAKS

// ✅ DO — Use data attributes for dynamic handlers
chip.dataset.code = leagueCode;
chip.addEventListener('click', function() { fn(this.dataset.code); });

// ✅ DO — Always wrap Firebase calls in try/catch
try {
  await setDoc(doc(db, 'collection', id), data);
} catch(e) {
  console.error('Save error:', e);
  showToast('Save failed', 'error');
}

// ✅ DO — Check auth before sensitive operations
if (!currentUser) { showToast('Sign in required', 'error'); return; }
```

### Naming Conventions
- Functions: `camelCase` — `generateLeague()`, `renderStandings()`
- Constants: `UPPER_SNAKE` — `ADMIN_EMAIL`, `FREE_PLAYER_LIMIT`
- IDs: `kebab-case` — `join-code`, `player-inputs`
- CSS classes: `kebab-case` — `match-card`, `btn-primary`

### Function Structure
- Each function does ONE thing
- Firebase functions are always `async`
- UI render functions are synchronous
- Always expose globals via `window.fnName = fnName`

## 4. Error Handling Standards

### User-Facing Errors
```javascript
// Always show user-friendly messages
showToast('Could not save — check your connection', 'error');

// Never show raw Firebase errors to users
// ❌ showToast(e.message, 'error');
// ✅ showToast('Save failed — try again', 'error');
```

### Firebase Error Codes to Handle
| Code | User Message |
|------|-------------|
| `permission-denied` | You don't have access to this |
| `unavailable` | You appear to be offline |
| `not-found` | League not found |
| `auth/user-not-found` | No account with this email |
| `auth/wrong-password` | Incorrect password |

### Console Logging
- `console.error()` — for actual errors (kept in production)
- `console.log()` — remove before production or use debug flag
- `console.warn()` — for non-critical warnings

## 5. Performance Standards

### Firebase Reads
- Use `onSnapshot` for real-time data (league scores)
- Use `getDoc` for one-time reads (history, profile)
- Always unsubscribe listeners when leaving: `if (unsubscribe) unsubscribe()`
- Limit queries: `.slice(0, 5)` for lists

### DOM Updates
- Batch DOM changes — build full HTML then insert once
- Use `innerHTML = ''` before rebuilding lists
- Avoid layout thrashing — don't read/write DOM alternately

## 6. Git Workflow

### Branch Strategy
- `main` — production, always deployable
- Feature branches for big changes: `git checkout -b feat/dupr-ratings`

### Commit Messages
```
feat: add DUPR rating system

```

### Pre-Push Checklist (automated by check.sh)
- [ ] `node --check app.v3.js` passes
- [ ] No broken `onclick` with `''` empty string patterns
- [ ] No duplicate `const` declarations
- [ ] `window.loginWithGoogle` exposed

### Deploy Process
```bash

git commit -m "..."
git push origin main   # Triggers Vercel auto-deploy
```

### Build Command
```
echo "window.__env = { FIREBASE_API_KEY: '$FIREBASE_API_KEY' };" > config.js
```

### Verify Deployment
```bash

curl -s https://pickleball-league-liard.vercel.app/app.v3.js | head -3
curl -s https://pickleball-league-liard.vercel.app/config.js
```

### Cache Busting
If Vercel serves stale files, add a comment to force cache invalidation:
```bash
echo "// $(date)" >> app.v3.js
git add . && git commit -m "fix: force Vercel cache bust" && git push
```

## 8. Firebase Standards

### Collections Structure
```
leagues/{leagueCode}
  - leagueCode, mode, players, teams, rounds
  - schedule (serialized as {r0:[], r1:[]})
  - results {matchId: {s1, s2, done}}
  - standings, isComplete, createdBy, updatedAt

history/{leagueCode}
  - same as leagues + completedAt, playedDate

groups/{userId}
  - list: [{name, players, memberEmails, memberUids, ownerUid}]

users/{userId}
  - uid, name, email, photo, lastSeen, plan

ratings/{docId}
  - name, rating, tier, lastUpdated
```

### Schedule Serialization
Firestore doesn't support nested arrays. Always serialize:
```javascript
// Serialize: array → object
function serializeSchedule(schedule) {
  const obj = {};
  schedule.forEach((round, i) => { obj['r' + i] = round; });
  return obj;
}
// Deserialize: object → array
function deserializeSchedule(obj, roundCount) { ... }
```

## 10. Known Issues & Workarounds

| Issue | Workaround |
|-------|-----------|
| Vercel caches old JS | Add `// date` comment to bust cache |
| Firebase Auth blocked | Use Firebase project API key, not Google Cloud |
| `node --check` passes but browser fails | Check for `''` empty quotes in onclick, literal newlines in regex |
| Groups visible to all users | Fixed — stored per `currentUser.uid` |
| Wife sees husband's league | Fixed — no auto-rejoin, per-user localStorage key |
